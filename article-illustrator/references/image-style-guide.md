# 配图风格指南

> 此为默认配图风格指南，article-illustrator 在为文章设计配图提示词时自动加载。用户可直接修改本文件，也可通过 skill 的复盘流程迭代更新。

---

## 一、整体调性

技术自媒体文章的配图风格定位：

- **科技感、简洁、信息可视化优先**：优先选择能辅助读者理解内容的图，而非纯粹装饰性的图
- **柔和明亮**：偏向浅色背景、低对比度、温馨色调（软彩蛋色/暖光），避免暗黑风或赛博朋克风
- **专业但不枯燥**：不要教科书风格的白底黑字图，需要有现代设计感

---

## 二、封面图设计原则

封面图是文章给读者的第一印象，需要：

**视觉构成**：
- 清晰的主视觉焦点（一个核心元素），不要塞太多内容
- 背景简洁，通常是深色渐变、抽象几何，或与主题相关的场景
- 整体有层次感，前景/中景/背景分明

**内容方向**：
- 抽象化地表达文章主题，不要字面插图（如：写 Agent 文章，不要画机器人，而是画"网络中相互协作的节点"）
- 科技感元素：代码流、数据可视化、网络连接、抽象几何（要求造型圆润或柔和）
- 氛围词汇：bright and airy, soft warm lighting, minimal, modern pastel aesthetic

**禁止**：
- 水印、logo（除非明确要求）
- 过于具象和卡通的人物形象
- 文字（封面图不应该包含文字，文字由平台层添加）

---

## 三、正文配图设计原则

正文配图的核心目标：**帮助读者理解，而不是装饰页面。**

**优先级排序**：
1. **信息图/流程图**：展示流程、架构、对比关系，最有价值
2. **概念可视化**：将抽象概念转化为具体的视觉比喻（如：将"缓存"比作"复印好的文件夹"）
3. **场景图**：烘托文章的情境和氛围（用于文章开头或转折处）
4. **纯装饰图**：最后选择，仅用于视觉节奏调整

**构图**：
- 留白充足，不要让画面显得拥挤
- 色调与封面保持一致的调性（同一篇文章的配图应有系列感）

**禁止**：
- 图中包含文字（英文 prompt 里要明确标注 `no text, no letters`）
- 截图风格（白底、UI 界面感）
- 与文章内容无关的纯装饰图

---

## 四、提示词结构模板

设计提示词时，遵循以下结构（英文输出）：

```
[主要内容描述], [风格关键词], [氛围关键词], [技术参数], no text, no watermark
```

**示例**：
```
Abstract visualization of interconnected AI agents as glowing nodes in a dark network, 
cinematic lighting, deep blue and purple color palette, minimal design, 
high contrast, professional tech illustration, no text, no watermark
```

---

## 五、颜色偏好

- **主色调**：米白色（`#f8f9fa`）、柔和拿铁色（`#fdfbf7`）、浅灰蓝（`#f1f5f9`）
- **强调色**：柔和蓝（`#93c5fd`）、莫兰迪紫（`#c4b5fd`）、蜜桃暖橘（`#fed7aa`）
- **禁用**：暗黑背景、赛博朋克高饱和霓虹色、纯红色（`#ff0000`）、荧光黄

---

## 六、禁止项汇总

| 禁止项 | 原因 |
|--------|------|
| 水印 / 版权标识 | 影响专业感 |
| 图中包含文字 | 与平台排版冲突 |
| 卡通/emoji 风格人物 | 与技术定位不符 |
| 白底截图风 | 过于平淡，无设计感 |
| 过于字面的主题插图 | 缺乏创意，显得廉价 |
